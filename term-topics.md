# Front-End Technology Seminar Topics

Below are example **term project topics** students may choose for their seminar project. Students may also propose their own topic with instructor approval.

These topics are intended for the **term-long research project**, not for weekly lightning talks. Weekly seminar topics will be listed separately.

A strong term project topic should:

- focus on a specific frontend technology, system, or architectural question
- include research
- include experimentation or a small prototype
- lead to a presentation explaining findings and trade-offs

---

## Architecture and Application Design

1. **Client State vs Server State** – When should data live in the browser and when should it be managed by the server?
2. **React Server Components** – How they work and how they change application architecture.
3. **Micro‑Frontend Architecture** – Breaking large applications into independently deployed frontends.
4. **Web Components vs React Components** – Interoperability, strengths, and weaknesses.
5. **Single Page Applications vs Multi‑Page Applications** – Modern trade‑offs.

---

## Performance and Optimization

6. **Frontend Performance Optimization Techniques** – Code splitting, lazy loading, and bundling.
7. **Image Optimization on the Web** – Formats, compression, and responsive images.
8. **Web Vitals and Real‑World Performance Metrics** – Measuring and improving user experience.
9. **Rendering Strategies** – Client rendering, server rendering, static generation, and streaming.
10. **Bundle Size and Dependency Management** – How large frameworks affect performance.

---

## CSS and Layout Systems

11. **Modern CSS Layout Systems** – Grid, Flexbox, and container queries.
12. **Design Systems in Practice** – Building reusable component systems.
13. **CSS Architecture Approaches** – BEM, utility CSS, and component‑based styles.
14. **Tailwind vs Traditional CSS** – Productivity, maintainability, and trade‑offs.
15. **CSS Animation vs JavaScript Animation** – Performance and developer workflow.

---

## Developer Tooling

16. **Modern JavaScript Build Tools** – Vite, Webpack, and the evolution of bundlers.
17. **TypeScript in Frontend Applications** – Benefits, costs, and developer experience.
18. **Testing Strategies for Frontend Applications** – Unit tests, integration tests, and end‑to‑end testing.
19. **Component Documentation Tools** – Storybook and design system documentation.
20. **Linting and Code Quality Tools** – ESLint, Prettier, and automated code review.

## Accessibility and User Experience

21. **Accessibility in Modern Web Apps** – ARIA, keyboard navigation, and screen readers.
22. **Inclusive Design Principles** – Designing interfaces for diverse users.
23. **Form Design and Usability** – Improving user input experiences.
24. **Dark Mode and Theming Systems** – Implementation patterns and UX implications.

## Proposing Your Own Topic

Students may propose their own term project topic with instructor approval.

A proposed topic should:

- investigate a specific frontend technology or architectural idea
- be framed as a **clear research question**
- include a **small experiment or prototype**
- be narrow enough to complete during the course

Topics that are too broad, too vague, or too speculative will need to be revised.

Examples:

**Too broad**

> React

**Better**

> How does Tailwind CSS affect development speed compared to component-scoped CSS?

**Stronger**

> Does Tailwind CSS reduce CSS bundle size in component-based React applications?

---

## Turning a Topic into a Project Question

Students are encouraged to turn a topic into a focused research question.

Example:

> How do React Server Components change the way data fetching is handled in modern web applications?

A strong project usually has three parts:

1. **Technology or concept**
2. **Research question**
3. **Experiment or prototype**

Example:

**Topic:** React Server Components  
**Question:** Do React Server Components significantly reduce JavaScript bundle size?  
**Experiment:** Compare bundle sizes between an RSC-based example and a standard React component approach.

A good term project should allow the student to:

- investigate real tools
- build a small experiment or prototype
- explain the strengths and weaknesses of the approach
