# Getting Started – ACS 4390

Welcome to **ACS 4390: Frontend Web Technology Seminar**. This course is designed differently from most programming classes. Instead of lectures and step‑by‑step labs, this class is organized as a **seminar and engineering lab** where students explore technologies together.

If this format feels unfamiliar at first, that is normal. This page explains how the course works and what you should do during the first week to get started successfully.

---

# How This Class Works

This course combines three activities that repeat each week:

1. **Lightning Talks** – short student presentations introducing technologies
2. **Discussion** – group analysis of tools, systems, and trade‑offs
3. **Engineering Lab** – experimentation and progress on your term project

The goal is not just to learn tools, but to understand **why technologies exist, what problems they solve, and when engineers should use them**.

---

# Weekly Rhythm

We meet **twice per week on Zoom**.

Each meeting has a different focus.

### Monday – Technology Exploration

Monday sessions introduce new ideas.

Typical activities:

- lightning talks
- technology discussions
- architecture analysis

These sessions explore the **frontend ecosystem** broadly.

---

### Wednesday – Engineering Lab

Wednesday sessions focus on **building and experimentation**.

Typical activities:

- prototype demos
- project progress updates
- technical problem solving

These sessions help you apply ideas from Monday discussions.

---

# Lightning Talks (Short Presentations)

Throughout the course students will give **lightning talks**.

A lightning talk is a **5–8 minute explanation** of a technology or concept.

A good lightning talk answers four questions:

1. What problem does this technology solve?
2. How does it work?
3. What are its advantages?
4. What are its trade‑offs?

Lightning talks are meant to introduce ideas quickly so the class can discuss them.

You will have time at the beginning of class to prepare your notes or slides.

---

# The Term Project

In addition to weekly discussions, you will complete a **term‑long project**.

Your project investigates a specific question about frontend technology.

Examples:

- Does Vite improve development performance compared to Webpack?
- What trade‑offs do React Server Components introduce?
- How do different state management libraries affect application complexity?

Projects should include:

- research
- a small prototype or experiment
- analysis of results

You will present your progress at the **midterm** and your final findings at the **end of the course**.

---

# Weekly Project Logs

To help track progress, you will submit a **short weekly project log**.

Each entry includes:

- what you learned
- what you built or tested
- what surprised you
- what you plan to try next

These logs help you stay on track and prepare for your midterm and final presentations.

# What You Should Do in Week 1

To get started in this course:

### 1. Read the course documents

Review the following pages:

- syllabus
- weekly class format
- weekly project log instructions

These explain how the seminar works.

---

### 2. Start thinking about project ideas

Your project should explore a **specific technical question**.

Good projects usually involve:

- comparing technologies
- measuring performance
- experimenting with frameworks
- analyzing architectural trade‑offs

You do not need a final idea yet, but you should begin thinking about possibilities.

---

### 3. Explore the frontend ecosystem

Spend some time looking at modern frontend tools such as:

- frameworks
- build tools
- state management libraries
- CSS systems

This exploration will help you choose a project topic.

---

### 4. Prepare to participate

Because this is a seminar, participation is essential.

You should be prepared to:

- speak during discussions
- share ideas
- explain concepts
- demonstrate experiments

Small classes work best when everyone contributes.

# The Goal of This Course

By the end of the semester you should be able to answer questions like:

- Why does a technology exist?
- What problem does it solve?
- What are its trade‑offs?
- When should engineers use it?

These are the kinds of questions experienced engineers ask when choosing tools and designing systems.

This course is designed to help you develop that level of thinking.
