

# Weekly Project Log

Throughout the course you will keep a **Weekly Project Log** to track your progress on your term project. The log helps you document what you are learning, what you are building, and what challenges you encounter along the way.

This log is meant to be **short, focused, and reflective**. Most entries should take **5–10 minutes** to complete.

The purpose of the project log is to:

- track your project progress
- help you organize your ideas
- document experiments and prototypes
- prepare for your midterm progress update and final presentation

---

# How Often to Submit

You will submit **one project log entry each week**.

Logs should be submitted before the **Wednesday Engineering Lab session** so we can discuss your progress in class.

---

# What to Include

Each weekly entry should include four short sections.

## 1. What I Learned

Briefly describe something new you learned while researching your topic.

Examples:

- a concept from an article
- a feature of a tool or framework
- an architectural idea

Example:

> I learned that Vite uses native ES modules in the browser during development, which allows it to start much faster than traditional bundlers.

---

## 2. What I Built or Tested

Describe any prototype, demo, or experiment you worked on.

This might include:

- a small demo application
- testing a library
- measuring performance
- comparing two technologies

Example:

> I built a small React project using Vite and measured the dev server startup time compared to a Webpack project.

---

## 3. What Surprised Me

Describe something unexpected, confusing, or interesting.

This section encourages curiosity and critical thinking.

Example:

> I expected Webpack to be much slower, but after caching the difference was smaller than I thought.

---

## 4. What I Will Try Next

Describe the next step for your project.

This might include:

- continuing the prototype
- testing another tool
- measuring results
- researching another source

Example:

> Next I want to measure the bundle size produced by Vite and compare it with Webpack.

---

# Example Entry

```
## Week 3 Project Log

Topic
React Server Components

What I Learned
React Server Components move data fetching to the server, which reduces the amount of JavaScript sent to the browser.

What I Built or Tested
I created a small Next.js project using React Server Components and compared it with a standard React component.

What Surprised Me
Server round-trips introduced some latency when fetching data.

What I Will Try Next
Measure bundle size and compare loading performance.
```

---

# Tips for Good Project Logs

- Keep entries short and clear
- Focus on what you actually tested or learned
- Document experiments even if they fail
- Use logs to prepare ideas for discussion in class

Your project log will become a **record of how your project developed over the course of the seminar** and will help you prepare for your **midterm progress update** and **final presentation**.
